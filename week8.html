本週課程介紹

恭喜你完成了前七週的學習！本週是期中複習週，我們將回顧前半學期所學的核心概念，並透過整合專案將這些知識融會貫通。

這週沒有新的理論內容，而是專注於複習、練習和實作。你將透過整合專案來實際應用所學的概念，並準備期中考試。

本週目標

  • 複習 Week 1-7 的核心概念
  • 整合多個主題的知識
  • 完成一個綜合性專案
  • 準備期中考試
  • 鞏固程式設計能力

一、核心概念複習

Week 1: 作業系統概論

重點回顧

  • 作業系統的定義與功能
  • 系統呼叫的使用（uname, getpid, getuid）
  • 作業系統的組成（核心、系統呼叫、工具程式）
  • 系統資源監控

Week 2: 行程管理與系統呼叫

重點回顧

  • 行程的概念與組成（PCB）
  • 行程狀態轉換
  • fork() 建立子行程
  • exec() 系列函數
  • wait() 和 waitpid()
  • 殭屍行程與孤兒行程

關鍵程式碼

pid_t pid = fork();
if (pid == 0) {
    // 子行程
    execl("/bin/ls", "ls", "-l", NULL);
} else {
    // 父行程
    wait(NULL);
}

Week 3: 行程排程

重點回顧

  • CPU 排程的目的
  • 評估標準：周轉時間、等待時間、回應時間
  • 排程演算法：
    • FCFS：簡單但可能有護航效應
    • SJF：最小平均等待時間
    • Priority：可能造成飢餓
    • Round Robin：公平，適合分時系統
  • 搶占式 vs 非搶占式

Week 4: 執行緒與並行程式設計

重點回顧

  • 執行緒 vs 行程
  • pthread 的使用：
    • pthread_create()
    • pthread_join()
    • pthread_exit()
  • 競爭條件問題
  • 執行緒池概念

關鍵程式碼

pthread_t thread;
pthread_create(&thread, NULL, thread_func, NULL);
pthread_join(thread, NULL);

Week 5: 行程同步與互斥

重點回顧

  • 臨界區域問題
  • 互斥鎖（Mutex）：
    • pthread_mutex_lock()
    • pthread_mutex_unlock()
  • 號誌（Semaphore）：sem_wait(), sem_post()
  • 條件變數（Condition Variable）
  • 經典問題：生產者-消費者、讀者-寫者
  • 死結的四個條件與預防

關鍵程式碼

pthread_mutex_lock(&mutex);
// 臨界區域
shared_variable++;
pthread_mutex_unlock(&mutex);

Week 6: 記憶體管理

重點回顧

  • 記憶體階層
  • 連續記憶體配置：
    • First Fit
    • Best Fit
    • Worst Fit
  • 碎片問題：內部碎片 vs 外部碎片
  • 分頁與分段
  • 動態記憶體管理：malloc, calloc, realloc, free

關鍵程式碼

int* array = (int*)malloc(10 * sizeof(int));
// 使用記憶體
free(array);

Week 7: 虛擬記憶體

重點回顧

  • 虛擬記憶體的優勢
  • 需求分頁（Demand Paging）
  • 頁面置換演算法：
    • FIFO：簡單但有 Belady's Anomaly
    • LRU：效能好但實作複雜
    • Optimal：理論最佳
  • TLB 的作用
  • 顛簸（Thrashing）

二、知識整合

2.1 概念關聯圖

主題 核心概念 實作技術 相關問題
行程管理 PCB、狀態轉換 fork, exec, wait 殭屍行程
排程 時間量子、優先權 FCFS, SJF, RR 飢餓、護航效應
並行 執行緒、共享記憶體 pthread 競爭條件
同步 臨界區域 Mutex, Semaphore 死結
記憶體 配置、碎片 First/Best/Worst Fit 外部碎片
虛擬記憶體 分頁、置換 FIFO, LRU 顛簸

2.2 常見陷阱與錯誤

程式設計常見錯誤

  • 忘記 pthread 程式要加 -pthread 編譯選項
  • fork() 後沒有正確處理父子行程
  • 使用 mutex 後忘記 unlock
  • malloc 後忘記 free（記憶體洩漏）
  • 競爭條件：多執行緒存取共享變數未加鎖
  • 死結：兩個執行緒互相等待對方釋放鎖

三、整合專案選項

選擇以下其中一個專案完成，建議投入 10-15 小時。

專案選項 1：多執行緒任務管理系統

專案描述

建立一個多執行緒的任務管理系統，模擬作業系統的排程和資源管理。

需求

  • 使用 pthread 建立工作執行緒池（至少 4 個執行緒）
  • 實作任務佇列，支援新增和移除任務
  • 實作至少兩種排程策略（如 FCFS 和 Priority）
  • 使用 mutex 和 condition variable 進行同步
  • 追蹤每個任務的等待時間和執行時間
  • 生成統計報告（平均等待時間、周轉時間等）

加分項

  • 支援動態調整執行緒池大小
  • 實作工作竊取（Work Stealing）
  • 視覺化顯示執行緒狀態
  • 死結偵測機制

專案選項 2：記憶體管理模擬器

專案描述

建立完整的記憶體管理模擬器，整合記憶體配置和虛擬記憶體。

需求

  • 實作動態記憶體配置（First Fit, Best Fit, Worst Fit）
  • 支援記憶體配置、釋放和壓縮
  • 實作分頁機制和頁表
  • 實作至少兩種頁面置換演算法（如 FIFO 和 LRU）
  • 模擬 TLB（至少 4 個項目）
  • 計算並顯示碎片、命中率、頁面錯誤率

加分項

  • 視覺化顯示記憶體狀態
  • 支援多個並行行程
  • 實作工作集追蹤
  • 顛簸偵測和報告

專案選項 3：簡易作業系統模擬器

專案描述

建立一個整合多個作業系統功能的模擬器。

需求

  • 行程管理：建立、終止、狀態查詢
  • 排程器：實作至少兩種排程演算法
  • 記憶體管理：實作簡單的記憶體配置
  • 同步機制：提供 mutex 和 semaphore
  • 互動式命令列介面
  • 支援基本命令（create, kill, ps, mem, help 等）

加分項

  • 實作簡單的檔案系統模擬
  • 支援行程間通訊（IPC）
  • 系統狀態視覺化
  • 執行腳本檔案

專案提交要求

必須包含

  • 完整的原始碼（含註解）
  • Makefile 或編譯說明
  • README 文件：
    • 專案簡介
    • 功能說明
    • 使用方式
    • 測試案例
  • 測試報告：
    • 測試案例和結果
    • 效能分析
    • 遇到的問題和解決方法
  • 設計文件：
    • 系統架構圖
    • 資料結構設計
    • 演算法說明
